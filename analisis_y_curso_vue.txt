--------------------------------------------------------------------------------
SECCIÓN 1: ANÁLISIS DETALLADO DE `Index.vue`
--------------------------------------------------------------------------------

El archivo `Index.vue` analizado es una vista de administración dentro de una aplicación
que utiliza Laravel, Inertia.js y Vue 3 con TypeScript. Su propósito principal es
gestionar una lista de "Propietarios", permitiendo buscar, ver, crear y editar registros
de propietarios de forma dinámica.

A continuación, se detalla qué se está usando y cómo se está usando en este componente:

1.1. ESTRUCTURA GENERAL (<script setup lang="ts">)
--------------------------------------------------
* ¿Qué es?: Es la sintaxis moderna de Vue 3 para trabajar con la Composition API.
* Cómo se usa:
  - `script setup`: Reduce el código boilerplate (código repetitivo). Todas las variables,
    funciones e importaciones declaradas aquí están disponibles automáticamente en la plantilla (<template>).
  - `lang="ts"`: Activa el soporte para TypeScript, permitiendo tipado estático, autocompletado
    y prevención de errores en tiempo de desarrollo.

1.2. IMPORTACIONES Y DEPENDENCIAS
---------------------------------
* `import { computed, nextTick, ref } from 'vue';`
  - `ref`: Para crear estados reactivos primarios.
  - `computed`: Para crear valores reactivos derivados (que se recalculan automáticamente al cambiar sus dependencias).
  - `nextTick`: Para esperar a que Vue termine de actualizar el DOM antes de ejecutar una acción.
* `import { Head } from '@inertiajs/vue3';`
  - Se usa para inyectar elementos en el `<head>` del HTML (como el título de la página) de manera dinámica.
* `import AuthenticatedLayout from '@/Layouts/AuthenticatedLayout.vue';`
  - El diseño base (Layout) de la aplicación para usuarios autenticados. Envuelve toda la página para mantener una estructura visual consistente (barra lateral, navegación, pie de página, etc.).
* Componentes Especializados:
  - `AdministradorHeader`, `AdministradorForm`, `AdministradorCardList`, `AdministradorTable`, `AdministradorEmptyState`.
  - Estos componentes modularizan la UI, separando el encabezado, el formulario, la vista en tabla (desktop), la vista en tarjetas (mobile) y el estado vacío.

1.3. PROPS Y TIPADO (defineProps)
---------------------------------
* Código:
  ```typescript
  const props = defineProps<{
      propietarios: Propietario[];
  }>();
  ```
* Cómo se usa:
  - `defineProps` es una macro de compilación (no necesita importarse). Define los datos que este componente recibe desde el controlador del backend (Laravel a través de Inertia.js).
  - Recibe un arreglo de objetos del tipo `Propietario`. Esto garantiza que los datos tengan la estructura correcta antes de ser renderizados.

1.4. ESTADO REACTIVO (ref)
--------------------------
* Variables reactivas declaradas:
  - `const search = ref('');`: Almacena el término de búsqueda escrito por el usuario en tiempo real.
  - `const showForm = ref(false);`: Un booleano que controla si el formulario de creación/edición está visible.
  - `const selectedPropietario = ref<Propietario | null>(null);`: Almacena el propietario que se va a editar. Si es `null`, significa que el formulario está en modo "crear nuevo".
  - `const formSection = ref<HTMLElement | null>(null);`: Una "Referencia de Plantilla" (Template Ref). Se vincula a un elemento del DOM en el template (`ref="formSection"`) para poder manipularlo directamente en el script.

1.5. PROPIEDADES COMPUTADAS (computed)
--------------------------------------
* Código:
  ```typescript
  const propietariosFiltrados = computed(() => {
      const termino = search.value.trim().toLowerCase();
      if (!termino) return props.propietarios;
      return props.propietarios.filter((propietario) => { ... });
  });
  ```
* Cómo se usa:
  - Filtra la lista original de propietarios recibida por props de manera reactiva y eficiente.
  - Si `search` está vacío, devuelve la lista completa.
  - Si hay texto, busca coincidencia en múltiples campos: `codigo_propietario`, `persona_id`, `carnet_identidad`, `nombre`, `apellido`, `telefono`, correo electrónico (`usuario?.email`), y el `estado`.
  - Utiliza el encadenamiento opcional `?.` para evitar errores si las relaciones `persona` o `usuarios` son nulas.
  - `filter(Boolean)` elimina los valores indefinidos o nulos del array antes de unirlos con `.join(' ')`.

1.6. EL FLUJO ASÍNCRONO Y nextTick
----------------------------------
* Código:
  ```typescript
  const desplazarAlFormulario = async () => {
      await nextTick();
      formSection.value?.scrollIntoView({
          behavior: 'smooth',
          block: 'start',
      });
  };
  ```
* Cómo se usa:
  - Cuando se hace clic en "Nuevo Propietario" o "Editar", `showForm.value` cambia a `true`.
  - Vue realiza las actualizaciones del DOM de forma asíncrona por rendimiento. Si intentáramos hacer scroll al elemento inmediatamente después de cambiar `showForm.value`, el formulario aún no existiría en el DOM (porque `v-if="showForm"` todavía no se habría evaluado).
  - Al usar `await nextTick()`, pausamos la ejecución de la función hasta que Vue actualice el DOM. Después, `formSection.value` ya no es `null` y podemos hacer un scroll suave (`scrollIntoView`) hacia el formulario.

1.7. EVENTOS Y MÉTODOS DE CONTROL
---------------------------------
* `nuevoPropietario()`: Prepara el formulario para creación (limpiando `selectedPropietario` a `null`), muestra el formulario y se desplaza hacia él.
* `editarPropietario(propietario)`: Recibe el propietario seleccionado, lo asigna a `selectedPropietario`, activa la visibilidad del formulario y se desplaza hacia él.
* `cerrarFormulario()`: Oculta el formulario y limpia la selección.
* `formularioGuardado()`: Reacciona al guardado exitoso cerrando el formulario.

1.8. ANÁLISIS DEL TEMPLATE (<template>)
---------------------------------------
* `<Head title="Propietarios" />`: Modifica el título de la pestaña del navegador a "Propietarios".
* `<AuthenticatedLayout>`: Envuelve todo el contenido. Define un slot llamado `#title` para pasarle el título "Propietarios" que se renderizará en la barra superior.
* `<AdministradorHeader ... />`:
  - `v-model:search="search"`: Vincula bidireccionalmente la variable `search` con la barra de búsqueda del componente hijo.
  - `:show-form="showForm"`: Envía la bandera de visibilidad.
  - `@create="nuevoPropietario"` y `@close="cerrarFormulario"`: Escucha eventos emitidos por el encabezado para abrir o cerrar el formulario.
* `<div ref="formSection">`: El contenedor del formulario. Contiene el atributo `ref` que permite a la función `desplazarAlFormulario` localizarlo físicamente en la pantalla.
* `<AdministradorForm v-if="showForm" ... />`:
  - Se renderiza de forma condicional (`v-if`).
  - Pasa el propietario seleccionado para determinar si está editando o creando.
  - Escucha `@saved` y `@cancel` para reaccionar tras guardar o cancelar.
* Condicional de Resultados:
  - Si `propietariosFiltrados.length > 0`: Muestra de forma complementaria la tabla (`AdministradorTable`) y la lista de tarjetas (`AdministradorCardList`) con los datos filtrados. Escuchan el evento `@edit` para abrir el editor.
  - Si no hay coincidencias: Renderiza el componente de estado vacío `<AdministradorEmptyState />` a través de la directiva `v-else`.

--------------------------------------------------------------------------------
MÓDULO 1: INTRODUCCIÓN Y FUNDAMENTOS (BÁSICO)
--------------------------------------------------------------------------------

1. ¿Qué es Vue.js?
Vue es un framework progresivo de JavaScript para construir interfaces de usuario.
Se centra en la capa de vista y es extremadamente fácil de integrar con otros
proyectos o frameworks (como Laravel a través de Inertia.js).

En Vue 3 existen dos formas de escribir componentes:
* Options API: Estilo antiguo (pero aún compatible), donde el código se organiza
  por opciones (`data`, `methods`, `computed`, `watch`).
* Composition API: Estilo moderno y recomendado (usado en tu proyecto). Se organiza
  por funciones lógicas, típicamente dentro de la etiqueta `<script setup>`.

2. Directivas Básicas de Vue
Las directivas son atributos especiales con el prefijo `v-` que aplican comportamiento
reactivo al DOM.

* v-bind (Atajos: `:`):
  Vincula reactivamente un atributo HTML a una variable de JavaScript.
  ```html
  <!-- Forma completa -->
  <img v-bind:src="urlImagen" />
  <!-- Atajo (Recomendado) -->
  <img :src="urlImagen" />
  ```

* v-model:
  Permite enlace de datos bidireccional (Two-Way Data Binding). Es ideal para
  elementos de formulario como inputs, textareas y selects.
  ```html
  <script setup>
  import { ref } from 'vue';
  const nombre = ref('');
  </script>
  <template>
    <input v-model="nombre" placeholder="Escribe tu nombre" />
    <p>Hola, tu nombre es: {{ nombre }}</p>
  </template>
  ```

* v-if, v-else-if, v-else:
  Renderizado condicional. Vue inserta o elimina físicamente el elemento del DOM
  según el valor de la expresión.
  ```html
  <p v-if="edad >= 18">Eres mayor de edad.</p>
  <p v-else>Eres menor de edad.</p>
  ```

* v-show:
  Alterna la visibilidad del elemento usando estilos CSS (`display: none`), pero
  el elemento permanece en el DOM. Útil para elementos que cambian de visibilidad
  con mucha frecuencia.

* v-for:
  Renderiza una lista de elementos basada en un array. Siempre requiere una clave
  única (`:key`) para ayudar a Vue a rastrear la identidad de cada nodo de forma eficiente.
  ```html
  <ul>
    <li v-for="item in lista" :key="item.id">
      {{ item.nombre }}
    </li>
  </ul>
  ```

* v-on (Atajo: `@`):
  Escucha eventos del DOM (como clicks, inputs, submits).
  ```html
  <!-- Atajo para escuchar click -->
  <button @click="incrementarContador">Incrementar</button>
  ```

3. Reactividad Básica: ref y reactive
Vue utiliza un sistema de reactividad basado en Proxies de ES6 para saber cuándo
cambian los datos y actualizar la pantalla automáticamente.

* ref:
  Se usa para declarar cualquier tipo de dato reactivo (primitivos como string,
  boolean, number, o también objetos/arrays).
  - En el `<script>`, para acceder o cambiar su valor debes usar `.value`.
  - En el `<template>`, se desenvuelve automáticamente (no uses `.value`).
  ```javascript
  import { ref } from 'vue';
  const contador = ref(0);
  contador.value++; // Incremento en script
  ```

* reactive:
  Se usa exclusivamente para objetos y arrays. No requiere `.value`, pero tiene
  limitaciones (como no poder reasignarse completamente y no poder desestructurarse
  sin perder la reactividad). Por regla general, en la comunidad se prefiere usar `ref`
  para todo por consistencia.

4. Clases y Estilos Reactivos
Vue facilita la asignación dinámica de clases y estilos CSS pasando objetos o arrays.
```html
<script setup>
import { ref } from 'vue';
const esActivo = ref(true);
</script>
<template>
  <!-- La clase "active" se aplicará solo si esActivo es true -->
  <div :class="{ active: esActivo, 'text-red': !esActivo }">
    Contenido
  </div>
</template>
```

--------------------------------------------------------------------------------
MÓDULO 2: COMPONENTES Y COMUNICACIÓN (INTERMEDIO)
--------------------------------------------------------------------------------

La base de cualquier aplicación en Vue es la división de la UI en componentes
pequeños, reutilizables e independientes.

1. Comunicación de Componentes: Props y Emits
Los componentes se comunican bajo el principio de "Props Down, Events Up" (Props
hacia abajo, eventos hacia arriba).

* defineProps (Entradas):
  El componente padre pasa datos al componente hijo mediante atributos.
  ```html
  <!-- Componente Hijo (TarjetaUsuario.vue) -->
  <script setup>
  defineProps({
      nombre: String,
      edad: Number
  });
  </script>
  <template>
      <div>{{ nombre }} - {{ edad }} años</div>
  </template>
  ```

* defineEmits (Salidas):
  El componente hijo emite eventos personalizados para notificar al padre sobre
  acciones o cambios.
  ```html
  <!-- Componente Hijo (BotonAlerta.vue) -->
  <script setup>
  const emit = defineEmits(['hacerClick']);
  function notificarPadre() {
      emit('hacerClick', 'Datos opcionales');
  }
  </script>
  <template>
      <button @click="notificarPadre">Hazme Click</button>
  </template>
  ```

2. Propiedades Computadas (computed)
Las propiedades computadas son funciones que procesan datos reactivos y devuelven
un nuevo valor.
* Tienen caché reactiva: Solo se vuelven a ejecutar si alguna de sus dependencias reactivas cambia.
* Son ideales para filtrados, ordenamientos, formateos complejos o cálculos matemáticos.
```javascript
const nombreCompleto = computed(() => {
    return `${nombre.value} ${apellido.value}`;
});
```

3. Observadores (watch y watchEffect)
A veces no queremos calcular un valor derivado (como computed), sino ejecutar
efectos secundarios (llamadas a APIs, guardar en localStorage, animaciones)
cuando una variable reactiva cambia.

* watch:
  Observa variables específicas y proporciona tanto el valor nuevo como el antiguo.
  ```javascript
  import { ref, watch } from 'vue';
  const buscar = ref('');

  watch(buscar, (nuevoValor, viejoValor) => {
      console.log(`Búsqueda cambió de ${viejoValor} a ${nuevoValor}`);
      // Aquí se puede hacer una llamada HTTP a una API
  });
  ```

* watchEffect:
  Ejecuta la función inmediatamente y rastrea automáticamente cualquier variable
  reactiva que se use dentro de ella de manera implícita.

4. Slots (Ranuras)
Los slots permiten pasar contenido HTML o componentes desde el padre al hijo,
en lugar de pasar solo datos planos.

* Slots por Defecto:
  ```html
  <!-- Hijo (Tarjeta.vue) -->
  <div class="card">
      <slot></slot> <!-- Aquí caerá el contenido -->
  </div>

  <!-- Padre -->
  <Tarjeta>
      <h3>Título de la tarjeta</h3>
      <p>Este es el cuerpo</p>
  </Tarjeta>
  ```

* Slots Nombrados (Named Slots):
  Permiten colocar contenido en múltiples ubicaciones específicas del componente hijo.
  Se definen con `name="..."` y se llenan en el padre con `v-slot:nombre` (o `#nombre`).
  ```html
  <!-- Hijo (Layout.vue) -->
  <header>
      <slot name="cabecera"></slot>
  </header>
  <main>
      <slot></slot>
  </main>

  <!-- Padre -->
  <Layout>
      <template #cabecera>
          <h1>Título de mi página</h1>
      </template>
      <p>Contenido principal</p>
  </Layout>
  ```

5. Ciclo de Vida del Componente (Lifecycle Hooks)
Cada componente de Vue pasa por una serie de fases de inicialización, montaje,
actualización y destrucción. Podemos engancharnos a estas etapas mediante funciones:

* `onMounted()`: Ejecutado justo después de que el componente ha sido insertado
  en el DOM real. Ideal para llamadas API o inicializar librerías externas.
* `onUpdated()`: Se ejecuta cuando el componente actualiza su DOM debido a un cambio de estado.
* `onUnmounted()`: Ejecutado antes de que el componente sea destruido. Se usa para
  limpiar temporizadores (setInterval), cancelar suscripciones o eliminar listeners globales.

--------------------------------------------------------------------------------
MÓDULO 3: REACTIVIDAD Y PATRONES AVANZADOS (AVANZADO)
--------------------------------------------------------------------------------

1. Referencias de Plantilla (Template Refs) y nextTick
Como vimos en `Index.vue`, en ocasiones es necesario interactuar directamente con
el DOM (por ejemplo para enfocar un input, controlar elementos multimedia o usar
el método `scrollIntoView`).

* Template Refs:
  Declaras una constante reactiva inicializada en `null` con el mismo nombre que
  el atributo `ref` en el HTML.
  ```html
  <script setup>
  import { ref, onMounted } from 'vue';
  const miInput = ref(null);

  onMounted(() => {
      miInput.value.focus(); // Enfoca el input automáticamente al cargar
  });
  </script>
  <template>
      <input ref="miInput" />
  </template>
  ```

* nextTick:
  Es una función global que devuelve una Promesa. Se resuelve después de que Vue
  haya actualizado el DOM tras un cambio de datos. Es crucial cuando dependes del
  tamaño, posición o existencia física de un elemento que acaba de cambiar por
  directivas reactivas como `v-if` o `v-show`.

2. Composables (Reutilización de Lógica)
En Vue 3, la mejor manera de reutilizar código que tiene estado reactivo es mediante
"Composables". Son funciones escritas con la Composition API que encapsulan lógica.
Por convención, sus nombres empiezan con "use" (como `useFetch`, `useAuth`, etc.).

Ejemplo de un Composable para rastrear las coordenadas del mouse:
```javascript
// composables/useMouse.js
import { ref, onMounted, onUnmounted } from 'vue';

export function useMouse() {
    const x = ref(0);
    const y = ref(0);

    function actualizar(event) {
        x.value = event.clientX;
        y.value = event.clientY;
    }

    onMounted(() => window.addEventListener('mousemove', actualizar));
    onUnmounted(() => window.removeEventListener('mousemove', actualizar));

    return { x, y };
}
```
Uso del Composable en cualquier componente:
```html
<script setup>
import { useMouse } from '@/composables/useMouse';
const { x, y } = useMouse();
</script>
<template>
    <p>El mouse está en: {{ x }}, {{ y }}</p>
</template>
```

3. Provide / Inject (Inyección de dependencias)
Cuando tenemos componentes muy anidados, pasar props a través de múltiples niveles
intermedios (conocido como Prop Drilling) se vuelve molesto.
`provide` e `inject` permiten al componente ancestro actuar como proveedor de
dependencias para todos sus descendientes, sin importar qué tan profundos estén.

* Componente Padre:
  ```javascript
  import { provide, ref } from 'vue';
  const tema = ref('oscuro');
  provide('temaGlobal', tema);
  ```
* Componente Nieto / Descendiente:
  ```javascript
  import { inject } from 'vue';
  const tema = inject('temaGlobal'); // Obtiene acceso reactivo a 'tema'
  ```

4. Teleport y Suspense
* Teleport:
  Permite renderizar una parte del template de un componente en un nodo del DOM
  diferente (fuera de la jerarquía de la aplicación, como por ejemplo al final
  de la etiqueta `<body>`). Es sumamente útil para ventanas modales, diálogos y popups
  para evitar problemas de CSS con `z-index` u `overflow: hidden`.
  ```html
  <Teleport to="body">
      <div class="modal">Contenido de la modal</div>
  </Teleport>
  ```

* Suspense:
  Un componente especial que maneja estados de carga para dependencias asíncronas
  en el árbol de componentes (componentes hijos que hacen peticiones asíncronas
  en su `<script setup>` usando `await` directo en el root).

5. Optimización y Rendimiento
* shallowRef / shallowReactive:
  A diferencia de `ref`, no convierte el objeto interno en reactivo de forma profunda.
  Solo rastrea si la propiedad `.value` cambia por completo. Es excelente para
  guardar instancias grandes de librerías externas (como mapas de Leaflet/Mapbox,
  editores de texto enriquecidos, gráficos tridimensionales de Three.js) donde la
  reactividad profunda arruinaría el rendimiento.

* v-once:
  Renderiza el elemento y componente una sola vez y omite las actualizaciones
  posteriores. Útil para optimizar contenido estático que nunca cambia.

* v-memo:
  Permite memorizar una parte del DOM. Vue omitirá la actualización de esa sección
  a menos que las variables especificadas en el arreglo de dependencias cambien.
  Muy útil en listas muy grandes de miles de filas con `v-for`.


--------------------------------------------------------------------------------
MÓDULO 4: ECOSISTEMA Y HERRAMIENTAS (PROFESIONAL)
--------------------------------------------------------------------------------

Para construir aplicaciones del mundo real robustas, Vue se acompaña de un ecosistema
de herramientas potentes.

1. Gestión del Estado Global: Pinia
Cuando los componentes necesitan compartir información compleja de forma global
(como el estado del usuario autenticado, carrito de compras, o configuraciones
de la app), se utiliza Pinia, el almacén de estados oficial para Vue 3.

Ejemplo de una Tienda (Store):
```javascript
// stores/counter.js
import { defineStore } from 'pinia';
import { ref, computed } from 'vue';

export const useCounterStore = defineStore('counter', () => {
    const count = ref(0);
    const doubleCount = computed(() => count.value * 2);
    function increment() {
        count.value++;
    }

    return { count, doubleCount, increment };
});
```

2. Vue con TypeScript
El uso de TypeScript en Vue mejora drásticamente la experiencia de desarrollo.
* Tipando Props:
  ```typescript
  defineProps<{
      titulo: string;
      activo?: boolean; // Prop opcional
  }>();
  ```
* Tipando Emits:
  ```typescript
  const emit = defineEmits<{
      (e: 'cambiar', id: number): void;
  }>();
  ```

3. Integración con Inertia.js (El enfoque de tu proyecto)
Inertia.js permite construir aplicaciones Single Page Applications (SPA) clásicas
sin la complejidad de crear una API REST o GraphQL completa.

* ¿Cómo funciona?:
  - Laravel maneja las rutas de la aplicación, controladores, bases de datos y la
    autenticación.
  - En lugar de retornar vistas Blade, el controlador de Laravel retorna una respuesta Inertia:
    `return Inertia::render('Administrador/Index', ['propietarios' => $propietarios]);`
  - Inertia se encarga de enviar estos datos al componente de Vue (`Index.vue`) como `props`.
  - Cuando navegas por la aplicación, Inertia intercepta los clicks a enlaces y realiza
    peticiones AJAX en segundo plano para obtener el JSON del nuevo componente,
    actualizando la vista sin recargar la página completa. Esto otorga la velocidad
    de una SPA con la simplicidad del desarrollo backend tradicional.
